# **Завдання 0: Домашнє завдання на самоопрацювання**

**Виконайте такі завдання:**

***Вам треба обрати систему на власний розсуд та сформувати систему даних для неї:***

1. Визначити, які дані необхідно зберігати.
2. Враховуючи користувачів системи, обрати, що є більш пріоритетним із CAP-теореми (якщо продукт складається з декількох систем, як, наприклад, інтернет-магазин, — для кожної складової продукту).
3. Використовуючи Back-of-the-Envelope техніку приблизно підрахувати, які обсяги даних необхідно підтримувати.
4. Базуючись на отриманій інформації, сформувати модель(і) даних системи, зокрема:
   - Який тип моделі даних використовувати (а можливо, поєднання яких типів)?
   - Яка буде схема даних (Schema-on-write або Schema-on-read)? Створити драфт сутностей та їхньої взаємодії.
   - Які use cases будуть покриватися (які операції будуть виконуватися)?
5. Використовуючи результати Back-of-the-Envelope розрахунків та доступної в мережі інформації, обрати конкретні технології (конкретну базу даних, яка імплементує обрану модель даних), що будуть задовольняти нефункціональні вимоги.

**Результати:**

1. Сформувати у вигляді UML-діаграм (наприклад, опис таблиць та їхніх схем, включно з взаємовідношеннями між таблицями, якщо була обрана реляційна модель, а також аргументація її вибору).
2. За наявності декількох моделей даних також зазначити, які будуть поєднувальні ланки між моделями (наприклад, інтернет-магазин передавав product_id і каталог до кошика, а потім — до OLTP).

**‼️Пам'ятайте:**

Не намагайтесь ускладнити систему. Достатньо розглянути декілька сутностей та відношення між ними, а також декілька шляхів використання системи (2-3). Не потрібно досконало розбирати систему до дрібниць.
